## 🧪 Testing Different Parameters

### Tone Variations
Try the same question with different tones:
- **Professional**: "Explain DeFi yield farming strategies"
- **Friendly**: "Can you help me understand yield farming in a casual way?"
- **Technical**: "Provide technical details on yield farming mechanisms"
- **Educational**: "Teach me about yield farming step by step"

### User Level Variations
Try the same question for different experience levels:
- **Beginner**: "What is a smart contract?" (user_level: beginner)
- **Intermediate**: "What is a smart contract?" (user_level: intermediate)  
- **Advanced**: "What is a smart contract?" (user_level: advanced)

### Domain Focus Testing
Try broad questions with specific domain focus:
- "Tell me about blockchain" (domain_focus: development)
- "Tell me about blockchain" (domain_focus: trading)
- "Tell me about blockchain" (domain_focus: security)

## 💡 Tips for Effective Testing

1. **Be Specific**: More specific questions yield better responses
2. **Provide Context**: Use the context parameter for personalized responses
3. **Try Different Levels**: Test the same question at different user levels
4. **Experiment with Tone**: Different tones can provide different perspectives
5. **Use Domain Focus**: Specify domain focus for targeted expertise
6. **Test Edge Cases**: Try unusual or complex scenarios
7. **Compare Responses**: Try the same question with different parameters

## 🔄 Iterative Testing

### Follow-up Questions
After getting a response, try follow-up questions like:
- "Can you explain that in more technical detail?"
- "What are the risks involved?"
- "How does this compare to traditional alternatives?"
- "What are some real-world examples?"

### Building Conversations
Use the chat history feature to build complex conversations:
1. Ask a basic question
2. Follow up with more specific questions
3. Request examples or clarifications
4. Explore related topics
